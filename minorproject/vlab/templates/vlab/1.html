Experiment 1:

Aim: Design a FA with ∑ = {0, 1} accepts those string which starts with 1 and ends with 0.

Theory:

States: To design the FA, you need to define the states. In this case, you'll have several states to represent different stages of processing the input string:

  • Q0: This is the initial state where the FA starts reading the input string.

  • Q1: This state indicates that the FA has read at least one '1' at the beginning of the string

  • Q2: This is an accepting state, indicating that the FA has read '1' at the beginning and '0' at the end of the string.

  • Q3: This is a dead state, representing an unrecoverable error state for invalid inputs. It ensures that the FA doesn't continue reading once an invalid input is encountered.

Transitions: Transitions define how the FA moves between states based on the input symbols ('0' and '1').

  • From Q0:
    On input '0', stay in Q0 to handle strings starting with '0'.
    On input '1', transition to Q1 to indicate that the string starts with '1'.

  • From Q1:
    On input '0', transition to Q2 to indicate that the string ends with '0'.
    On input '1', stay in Q1 to allow for additional '1's at the beginning.

  • From Q2:
    On input '0', stay in Q2 to accept additional '0's at the end.
    On input '1', transition back to Q1 to handle strings like "101" where '1' appears after '0'.

  • From Q3 (Dead State): On input '0' or '1', stay in Q3 to handle invalid inputs.

These test cases cover various scenarios to ensure that the FA correctly accepts strings that start with '1' and end with '0', while rejecting strings that do not meet this criteria.

Conclusion:

In conclusion, this FA has been designed to recognize strings that start with '1' and end with '0' by transitioning through states based on the input symbols. If the FA reaches the accepting state Q2, it means the input string satisfies the specified criteria and should be accepted. If it reaches the dead state Q3, it means the input is invalid, and the FA cannot proceed further.
