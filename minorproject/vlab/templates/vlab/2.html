Experiment 2:

Aim: Draw a DFA for the language accepting strings ending with ‘00’ over input alphabets ∑={0, 1}

Theory:

States: To design the DFA, you need to define the states. In this case, you'll have three states to represent different stages of processing the input string:

  • Q0: This is the initial state where the DFA begins processing the input string.
  • Q1: This state indicates that the last symbol read was '0' but not two consecutive '0's.
  • Q2: This is an accepting state, indicating that the DFA has successfully read '00' at the end of the string.

Transitions: Transitions define how the DFA moves between states based on the input symbols ('0' and '1').

  • From Q0:
    On input '0', stay in Q0 (to handle strings not ending with '00').
    On input '1', transition to Q0 (to continue scanning for '00').
  • From Q1:
    On input '0', transition to Q2 (to indicate the string ends with '00').
    On input '1', transition to Q0 (to continue scanning for '00').
  • From Q2:
    On input '0' or '1', stay in Q2 (to continue accepting '0's at the end).

Procedure:

(using the simulator)

Test Cases:

  • Test Case 1: String ends with '00'
    Input: "101100"
    Expected Outcome: Accept (The input string ends with '00'.)

  • Test Case 2: String ends with '00' but contains '1's in the middle
    Input: "11011000"
    Expected Outcome: Accept (The input string ends with '00' but contains '1's in the middle.)

  • Test Case 3: String does not end with '00'
    Input: "1010101"
    Expected Outcome: Reject (The input string does not end with '00'.)

  • Test Case 4: String contains only '0's and ends with '00'
    Input: "0000"
    Expected Outcome: Accept (The input string contains only '0's and ends with '00'.)

  • Test Case 5: String contains only '1's
    Input: "111111"
    Expected Outcome: Reject (The input string does not end with '00'.)

  • Test Case 6: Empty String
    Input: ""
    Expected Outcome: Reject (The empty string does not meet the condition.)

  • Test Case 7: Single '0'
    Input: "0"
    Expected Outcome: Reject (The input string is too short to meet the condition.)

  • Test Case 8: Single '1'
    Input: "1"
    Expected Outcome: Reject (The input string is too short to meet the condition.)

  • Test Case 9: String with spaces and other characters
    Input: "0 0 ! 1 0 0"
    Expected Outcome: Accept (The input string ends with '00', ignoring spaces and other characters.)

  • Test Case 10: Long string with valid end
    Input: "10101010101000"
    Expected Outcome: Accept (The input string ends with '00'.)

These test cases cover various scenarios to ensure that the DFA correctly accepts strings that end with '00' and rejects strings that do not meet this condition.

Conclusion:

In summary, this DFA has been designed to recognize strings that end with '00' by transitioning through states based on the input symbols. If the DFA reaches the accepting state Q2, it means the input string satisfies the specified criteria and should be accepted. If it does not reach Q2, it indicates that the string does not end with '00', and the DFA rejects it.
