Experiment 3:

Aim: Design FA with ∑ = {0, 1} accepts even number of 0's and even number of 1's.

Theory:

States: To design the FA, you need to define the states. In this case, you'll have several states to represent different parity (even or odd) counts of '0's and '1's in the input string. We will define the states as follows:

  • Q0: Initial state. Represents an even number of '0's and an even number of '1's encountered so far.
  • Q1: Represents an odd number of '0's and an even number of '1's encountered so far.
  • Q2: Represents an even number of '0's and an odd number of '1's encountered so far.
  • Q3: Represents an odd number of '0's and an odd number of '1's encountered so far.

Transitions: Transitions define how the FA moves between states based on the input symbols ('0' and '1') and the desired behavior of the FA:

  • From Q0:
    On input '0', transition to Q1 (to represent an odd number of '0's).
    On input '1', transition to Q2 (to represent an odd number of '1's).
  • From Q1:
    On input '0', transition to Q0 (to return to even '0's).
    On input '1', transition to Q3 (to represent an odd number of '0's and '1's).
  • From Q2:
    On input '0', transition to Q3 (to represent an odd number of '0's and '1's).
    On input '1', transition to Q0 (to return to even '1's).
  • From Q3:
    On input '0', transition to Q2 (to return to even '0's).
    On input '1', transition to Q1 (to return to even '1's).

Test Cases:

  • Test Case 1: Even number of '0's and '1's
    Input: "001100"
    Expected Outcome: Accept (The input string contains two '0's and two '1's, which is even for both.)

  • Test Case 2: Odd number of '0's and even number of '1's
    Input: "0110"
    Expected Outcome: Reject (The input string contains three '0's, which is odd for '0's.)

  • Test Case 3: Even number of '0's and odd number of '1's
    Input: "10011"
    Expected Outcome: Reject (The input string contains three '1's, which is odd for '1's.)

  • Test Case 4: Empty String
    Input: ""
    Expected Outcome: Accept (The empty string contains an even number of both '0's and '1's, which is 0 for both.)

  • Test Case 5: String with only '0's
    Input: "000000"
    Expected Outcome: Accept (The input string contains six '0's, which is even for '0's.)

  • Test Case 6: String with only '1's
    Input: "111"
    Expected Outcome: Accept (The input string contains three '1's, which is even for '1's.)

  • Test Case 7: Alternating '0's and '1's
    Input: "010101"
    Expected Outcome: Accept (The input string contains three '0's and three '1's, which is even for both.)

  • Test Case 8: Long string with even numbers
    Input: "00001111000011110000"
    Expected Outcome: Accept (The input string contains an even number of '0's and an even number of '1's.)

  • Test Case 9: Single '0' and Single '1'
    Input: "01"
    Expected Outcome: Reject (The input string contains an even number of '0's but an odd number of '1's.)

  • Test Case 10: String with spaces and other characters
    Input: "0 1 1 0"
    Expected Outcome: Accept (The input string contains an even number of '0's and an even number of '1's, ignoring spaces and other characters.)

These test cases cover various scenarios to ensure that the FA correctly accepts strings with an even number of '0's and an even number of '1's while rejecting strings that do not meet this criteria.

Conclusion:

In conclusion, we have designed a finite automaton (FA) that accepts strings with an even number of both '0's and '1's over the input alphabet ∑ = {0, 1}. The FA maintains four states, representing different parity combinations of '0's and '1's, and transitions between these states based on the input symbols. All states in this FA are accepting states because they represent even parity.
