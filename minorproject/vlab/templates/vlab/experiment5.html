Experiment NO.5
Aim: Draw a deterministic finite automaton which either starts with 01 or ends with 01 of a string containing 0,1 in it.

Theory

States: To design the deterministic finite automaton (DFA) that recognizes strings that either start with '01' or end with '01' but not both, we can create a DFA with distinct states and transitions for these conditions. The alphabet is ∑ = {0, 1}.


  • Q0: Initial state.
  • Q1: State to indicate that the string starts with '0'.
  • Q2: State to indicate that the string starts with '01' and can continue with '0's.
  • Q3: State to indicate that the string starts with '01' and can continue with '1's.
  • Q4: Accepting state to indicate that the string ends with '01'.
  • Q5: Dead state to handle invalid inputs.

Transitions: Define the transitions between states based on the input symbols ('0' and '1') and the desired behavior of the DFA.


  • From Q0:
    On input '0', transition to Q1 (to indicate that the string starts with '0').
    On input '1', stay in Q0 (to handle strings starting with '1').
  • From Q1:
    On input '0', transition to Q2 (to indicate that the string starts with '01').
    On input '1', stay in Q0 (to handle strings starting with '10' or '11').
  • From Q2:
    On input '0', stay in Q2 (to continue accepting '0's after '01').
    On input '1', transition to Q3 (to indicate that the string starts with '01' and can continue with '1's).
  • From Q3:
    On input '0', stay in Q2 (to handle strings like "010101" where '01' can continue with '0's).
    On input '1', stay in Q3 (to continue accepting '1's after '01').

Test Cases:

These test cases cover various scenarios to ensure that the DFA correctly accepts strings that either start with '01' or end with '01' but not both. It also rejects strings that do not meet these conditions.

Conclusion

In conclusion, we have designed a deterministic finite automaton (DFA) that recognizes strings meeting the criteria of either starting with '01' or ending with '01', but not both simultaneously. The DFA has distinct states and transitions to handle these conditions, along with a dead state to handle invalid inputs.
